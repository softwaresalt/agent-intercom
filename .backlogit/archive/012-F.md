Security upgrade of the `rmcp` MCP SDK from 0.13.0 to >=1.4.0 to remediate RUSTSEC-2026-0189 — a DNS rebinding vulnerability in rmcp's Streamable HTTP server transport (CVSS 8.8, HIGH; published 2026-04-29).

## Context
- The vulnerability affects the exact transport this server uses (`transport-streamable-http-server`, mounted at `/mcp`).
- Fix requires `rmcp >=1.4.0`, a breaking major version bump. rmcp 0.13→1.4 changes the ServerHandler/ToolRouter and transport APIs (similar in scope to the prior 0.5→0.13 upgrade tracked as 005.010-T).
- `Cargo.toml` already carries an `rmcp-upgrade` feature-gate placeholder (US5) anticipating this work.
- The advisory is temporarily ignored in `.cargo/audit.toml` (RUSTSEC-2026-0189) so CI audit passes; risk accepted because the project is pre-release and not publicly deployed.

## Acceptance criteria
- `rmcp` upgraded to a version >=1.4.0 that no longer triggers RUSTSEC-2026-0189.
- Transport/handler/tool-router code migrated to the new rmcp API; `cargo build`, `cargo clippy --all-targets -- -D warnings -D clippy::pedantic`, and `cargo test` all pass.
- The `RUSTSEC-2026-0189` ignore entry is removed from `.cargo/audit.toml` and `cargo audit` passes without it.
- Remove/retire the `rmcp-upgrade` feature-gate placeholder once complete.
