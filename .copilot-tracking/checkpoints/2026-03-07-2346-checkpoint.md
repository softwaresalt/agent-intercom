## Key Decisions

1. **`parse_prompt_type` uses 2-arm match** — `"clarification"`, `"error_recovery"`, `"resource_warning"` explicit; `_` wildcard covers `"continuation"` AND all unknown/empty (clippy::match_same_arms prevents explicit "continuation" arm)
2. **`prompt_id` used as `prompt.id`** — same pattern as Phase 3's `request_id → approval.id`; enables ACP JSON-RPC correlation and `prompt_repo.get_by_id(prompt_id)` in Slack handler
3. **`slack_ts` NOT stored on prompt record** — intentional asymmetry vs. clearance; Slack prompt handler extracts ts from event payload (`message.origin.ts`), not DB
4. **D2 conditional posting** — `post_message_direct` when `thread_ts=None` (captures ts for thread anchor); `enqueue` when thread exists (ordered delivery)
