## Key Decisions

1. **D1: Shared Block Builders** — Extract build_approval_blocks, approval_buttons, build_prompt_blocks, prompt_buttons, helpers from MCP tool files to src/slack/blocks.rs as pub(crate).
2. **D2: Conditional Posting** — Clearance always uses post_message_direct. Prompts use direct post when creating first thread (thread_ts=None), enqueue when thread exists.
3. **D3: Log and Continue** — Event handler errors emit warn! and skip to next event. No panic/unwrap.
4. **D4: No Oneshot Channels** — ACP handlers register with AcpDriver directly; Slack button handlers resolve via polymorphic state.driver.
5. **D5: Driver Access via Trait Object** — Handlers use state.driver (Arc<dyn AgentDriver>) directly, no downcast needed.
6. **UF-03 Fix: DB-First Ordering** — T010 persists to DB first, then registers driver with DB-generated approval.id (not event.request_id) to ensure Slack button action_ids match driver lookup keys.
7. **SC-003 Amended** — Changed from "100% persistence" to "attempted persistence with bounded failure handling" to reconcile with D3 log-and-continue.
