## Key Decisions

1. **Programmatic API-level testing over browser-based Playwright**: The backlog mentioned "Playwright or equivalent" but Slack interactions are API-based (Socket Mode, Block Kit payloads), not browser-based. Spec correctly scopes to testing the slack-morphism type layer programmatically.
2. **No external dependencies**: All Slack UI tests must run without credentials, network, or human operator (FR-007, SC-005). This ensures CI compatibility.
3. **Test existing `blocks.rs` functions in isolation**: The Block Kit builders are pure functions taking domain models and returning `SlackBlock` structures — ideal for unit-level assertion.
4. **Simulate interactions via synthetic payloads**: Rather than mocking the entire Slack API, tests construct `SlackInteractionEvent` payloads and dispatch them through the existing event handler pipeline.
