## Key Decisions

1. Add a new automation-specific agent and skill instead of changing `hitl-test`, so manual operator-driven coverage remains available as a fallback.
2. Build the first browser automation layer as a self-seeding Playwright suite because the pre-existing visual specs mostly assume Slack messages or threads already exist in the channel.
3. Reuse existing Rust test commands and the `tests\live` / `tests\visual` infrastructure rather than introducing a new Rust binary for orchestration.
4. Make the harness report each phase as PASS, FAIL, or SKIP so missing credentials and missing Playwright dependencies are explicit instead of ambiguous.
5. Treat the Copilot `write EAGAIN` issue as an external transport problem until logs prove otherwise, because no `EAGAIN` path was found in the repository itself.

## Failed Approaches

- Considered wrapping the existing Playwright Slack UX suite unchanged, but abandoned that approach after reviewing the specs and finding that most scenarios depend on pre-existing Slack state and therefore would not remove the manual HITL dependency.
