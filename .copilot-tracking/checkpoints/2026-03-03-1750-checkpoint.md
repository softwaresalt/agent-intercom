## Key Decisions

1. **run_reader signature change**: Added `max_msg_rate: u32` parameter (pass 0 to disable). Required updating 2 test callers and 1 production caller in commands.rs.
2. **StallDetector refactor**: Used `next_threshold` mutable variable initialized to `inactivity_threshold.saturating_sub(initial_elapsed)` and reset to `inactivity_threshold` after first iteration. Cleaner than pre-loop section which would have duplicated Stall event emission.
3. **T151 ordering**: After moving `update_status(Active)` into the `if acp_driver` branch, added a separate activation in the `else` branch, then re-fetched session record via `get_by_id` after the if/else to avoid multiple code paths returning different `active` values.
4. **T154 workspace mapping**: Used `workspace_mappings` (hot-reload RwLock) instead of `config.workspace_root_for_channel()` (static config). WorkspaceMapping fields are `channel_id: String` (not `slack_channel_id`) and `path: Option<PathBuf>` (not `workspace_root`).
5. **SessionMode fix**: `SessionMode::Direct` doesn't exist — must use `SessionMode::Local` in slack_client_tests.rs.

## Failed Approaches

- **Pre-loop initial_elapsed handling**: Initially added a pre-loop section in `StallDetector::run()` to handle the first wait separately. This would have emitted two Stalled events when `initial_elapsed >= threshold`. Replaced with `next_threshold` mutable variable approach.
- **T154 field names**: Initially used `slack_channel_id` and `workspace_root` for WorkspaceMapping. Actual fields are `channel_id` and `path`.
