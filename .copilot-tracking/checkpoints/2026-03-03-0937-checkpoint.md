## Key Decisions

1. **`parse_checkpoint_args` made `pub` (not `pub(crate)`)**: External test crates cannot access `pub(crate)` items. Making it `pub` was necessary for unit tests in `tests/unit/`. This is the right trade-off: a parsing utility function is not security-sensitive.

2. **ACP session guard as outermost axum middleware layer**: Guard runs first (before accept-header normalization) so invalid sessions are rejected without wasting resources on header processing. Guard reads `session_id` directly from URI, not from `PendingParams`.

3. **`resolve_command_session` fallback to interrupted**: Finds interrupted sessions ordered by `updated_at DESC` (most recent first). Falls back only when `find_active_by_channel` returns empty — preserves existing behavior for normal flows.

4. **T118 and T121 deferred**: T118 requires Slack message mocking infrastructure that doesn't exist yet. T121 requires a new Block Kit action handler (callback routing) which is Phase 15 scope. `/arc session-cleanup` provides the same operator capability.

5. **`CARGO_TARGET_DIR=target\test-build`**: Used to bypass locked `agent-intercom.exe` (held by two running debug server processes). This is a Windows-specific workaround; tests pass correctly in the alternate build dir.

## Failed Approaches

- **`serve_http` in integration test helper**: Initial test used `serve_http` which internally calls `bind_http` (re-binds to port 0), losing track of the actual port. Fixed by using `serve_with_listener` with a pre-bound listener instead.
- **Drop-listener race condition**: First attempt dropped the listener before `serve_http` re-bound it, causing port races. Fixed by NOT dropping the listener and passing it directly to `serve_with_listener`.
